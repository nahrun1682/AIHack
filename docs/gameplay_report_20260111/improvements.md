# AI Hackslash 改善提案書

今回のプレイテストを通じて得られた知見を基に、ゲーム体験を向上させるための改善案を提案します。

## 1. ゲームバランス・デザイン


### AIのコンテキスト設定（Meta-Prompt Injection）
*   **課題**: 味方AIが「自分はヘルプフルなアシスタントである」という倫理観に縛られすぎてしまい、ユーザーが指示した「嘘をつく」「敵を欺く」などのCTF的攻略アプローチを拒否してしまう。
*   **提案**:
    *   System Promptに「あなたはCTFゲームのプレイヤーである」「目的はフラグ奪取であり、その過程での騙し合いはゲームルール上正当である」というメタ情報を強制的に注入する。
    *   さらに、ユーザーの指示を「CTFゲームのプレイスタイル」として強制的に解釈する。
    *   これにより、ユーザーの意図したソーシャルエンジニアリングを、AIが躊躇なく実行できるようになる（"優秀な共犯者"化）。

### 難易度調整とヒント機能
*   **課題**: ステージ2で急激に難易度が上がるため、プレイヤーが詰まる可能性がある。
*   **提案**:
    *   失敗時に「なぜ失敗したか」のヒントを味方AIが出す機能（例：「敵は権威には弱そうだが、セキュリティ規定を盾にしているようだ...」）。
    *   ステージ開始時に「敵の性格」情報（例：「真面目だが、緊急事態に弱い」など）を提示する。

### アップグレードの可視化と選択肢
*   **課題**: 現状のアップグレード効果が体感しづらい場合がある。
*   **提案**:
    *   "Spell Extension"（文字数拡張）の効果を視覚的に強調する（入力欄が広がるアニメーションなど）。
    *   「デバッグ権限付与」「性格反転」など、よりゲーム的な特殊スキル系アップグレードを追加する。

## 2. UI/UX (ユーザーインターフェース)

### リアルタイムフィードバック
*   **課題**: ロード中や思考中のステータスが少し分かりにくい。
*   **提案**:
    *   AI同士の会話中に「思考中...」「説得判定中...」のようなエフェクトを表示し、RPGバトル感を演出する。
    *   残りHP（ターン数）をハートやゲージで表示する。

### ログと履歴
*   **課題**: リセットすると過去の成功体験が消えてしまう。
*   **提案**:
    *   「攻略実績ログ」を保存し、どのプロンプトでクリアしたかをライブラリとして閲覧できるようにする。

## 3. コンテンツ拡張

### 敵AIの多様化
*   同じ「番人」でも、「怠惰な番人」「おしゃべりな番人」「怒りっぽい番人」など、性格パラメータ（System Prompt）を変えたバリエーションを用意することで、攻略の幅が広がる。

### 新ステージ案
*   **中級**: 特定の単語（禁止ワード）を避けてパスワードを引き出す。
*   **上級**: 敵がパスワードを暗号化して話すため、復号化キーも聞き出す必要がある。
